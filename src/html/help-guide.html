Help Guide

Getting Started with Wordmark

Wordmark is an open source AI assistant for the OpenAI and xAI Responses APIs and local LM Studio or Ollama servers, all while keeping your data private and stored locally on your device.

Use Philosophy: Not a Companion

Wordmark is a creative and technical tool — not a buddy, therapist, or romantic partner. If you’re looking for a “companion,” you’re in the wrong place and the maintainer frowns on that usage. Please use Wordmark to build, learn, and make things — not to simulate intimacy. Read the short policy here: Not a Companion.

Connecting Providers

To get the most out of Wordmark, connect it to OpenAI's or xAI's API, or point it at a local LM Studio or Ollama server. Hosted providers require an API key, while LM Studio and Ollama run on your machine and only need their base URLs.

Why You Need Credentials

API keys are digital passwords that authorize requests you send to a hosted service. Wordmark stores the OpenAI and xAI keys securely in your browser's local storage and never sends them anywhere except to the endpoints you target. LM Studio and Ollama do not use API keys—the app simply needs to know where your local server is listening.

How to Connect

OpenAI API Key

  1. Visit the OpenAI API Keys page
  2. Create an account or sign in if you don't already have one
  3. Click on "Create new secret key"
  4. Give your key a name (e.g., "Wordmark")
  5. Copy the API key and paste it into Wordmark's settings

xAI API Key

  1. Visit the xAI Console
  2. Create an account or sign in with your existing credentials
  3. Open the API Keys section and click "Create new key"
  4. Copy the generated key (it will only be shown once)
  5. Paste the key into Wordmark's Settings → API Keys → xAI field

LM Studio Base URL

  1. Download LM Studio for your platform and install it
  2. Launch LM Studio and start the local server (default is http://localhost:1234)
  3. Load or download a chat-capable model in LM Studio so it is exposed via the /models endpoint
  4. In Wordmark, open Settings → API Keys → LM Studio
  5. Enter the base URL (without the trailing /v1) and click “Save LM Studio URL”

Ollama Server

  1. Install Ollama for your platform
  2. Start the Ollama server (default is http://localhost:11434)
  3. Pull at least one model so it appears in the model list (for example, ollama pull qwen3)
  4. In Wordmark, open Settings → Model and choose the Ollama provider

Managing Your API Keys in Wordmark

  1. Click the Settings icon in the top right corner
  2. Navigate to the "API Keys" section
  3. Enter your API keys in the appropriate fields
  4. Click "Save API Keys" to store them securely in your browser

Security Note: Your API keys are stored only in your browser's local storage and are only sent to their respective services when making API calls. Wordmark does not store your keys on any servers.

Choosing and Using AI Models

Wordmark gives you access to hosted OpenAI and xAI models as well as local models served through LM Studio or Ollama.

Available AI Providers

  • OpenAI: GPT-series models for hosted, general-purpose conversation and coding tasks
  • xAI: Grok models accessed via the xAI Responses-compatible API
  • LM Studio: Local, open-source models you run on your own hardware
  • Ollama: Local models served from your Ollama runtime

Using Advanced Features

Tools and Extensions

Wordmark supports a small set of tools that extend the AI's capabilities:

  • Web Search: Let the AI use OpenAI-managed web search for fresh information when the OpenAI service is active.
  • Weather Information: Get short forecasts via the built-in Open-Meteo helper.
  • Custom MCP Servers: Register your own MCP endpoints to unlock additional domain-specific tools.

Note: The built-in helpers do not require extra keys. MCP servers manage their own authentication and may need credentials on their side.

Memory

Use the Memory feature to let the assistant remember or forget brief details you provide.

  • Enable Memory: Open Settings → Memory tab and switch on “Enable Memory”.
  • Limit: Choose how many items to retain (default 25). Oldest items are removed first when the limit is exceeded.
  • Remember: When you say “remember …”, the assistant may call a special function to store a short summary of that detail.
  • Forget: Ask to “forget …” to remove a matching memory. The assistant uses a case-insensitive keyword match.
  • Context: When enabled, your saved memories are appended to the system prompt as a bullet list for better personalization.
  • Local Only: All memories are stored in your browser (no server). You can clear or delete individual items at any time.

Text-to-Speech

Wordmark can read responses aloud using text-to-speech technology. Click the speaker icon next to a message to hear it spoken. Requires an OpenAI API key.

Theme Customization

Personalize your experience by choosing from various themes in the settings panel, including Dark, Light, Metal, Neon, and more.

Data Privacy and Storage

Wordmark is designed with privacy in mind:

  • All your conversations and settings are stored locally in your browser
  • No data is sent to any servers except the AI provider you choose to use
  • You can export your conversation history or clear it at any time
  • When using local LM Studio or Ollama models, no data leaves your device at all

Troubleshooting

Common Issues

  • API Key Errors: Double-check that your API key is entered correctly and hasn't expired
  • Model Not Available: Ensure you have the appropriate API key for the model you're trying to use
  • LM Studio Not Connecting: Verify that LM Studio is running locally and that the server URL in settings matches the port showing inside LM Studio
  • Ollama Not Connecting: Verify that Ollama is running locally and listening on http://localhost:11434
  • Missing Tool Results: Check that you've provided any necessary API keys for specific tools
